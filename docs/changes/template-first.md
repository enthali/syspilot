## Summary

Separate syspilot's distributed agent files from its own dogfooding installation. Introduce a `templates/agents/` directory containing generic, language-agnostic skeleton agents (the **product**), while `.github/agents/` becomes syspilot's own project-specific installation (the **consumer**). The Setup Agent's source path changes from `.github/agents/` to `templates/agents/`. The release workflow validates the distribution path by installing from templates into `.github/`.

This addresses the fundamental architectural tension: the repo is both the product AND a user of the product. Currently the same files serve both roles, leading to Python/Sphinx-specific agents being distributed to non-Python projects.

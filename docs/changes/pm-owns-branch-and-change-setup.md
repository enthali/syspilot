## Summary

Clarify and enforce the ownership boundary between PM and CM for branch creation, Change Document initialization, and integration. PM creates the feature branch and the initial Change Document (from template) before sending the CR. CM receives a ready-made branch and document and fills in the spec/engineering levels. Integration into `development` is the responsibility of the PM role — CM signals readiness, the integrator performs the merge. Feature branches are retained after merge for forensic/bisect purposes and cleaned up by the Release Agent at release time. Motivation: without explicit ownership of branch and document setup, agents must infer conventions from context, which is fragile; explicit structural ownership eliminates this class of ambiguity. Acceptance: (1) PM agent describes branch + CD creation duty, (2) CM agent states CM receives branch + CD from PM and never creates them, (3) CM agent states CM never merges to `development` — Integration is a PM-role responsibility, (4) feature branches are not deleted after merge, (5) Release Agent file describes the cleanup step.
